PROMPT 7E — Registro de usuario (aclaración de roles)

[VISTA REGISTRO DE USUARIO — LÓGICA DE ROLES]
Adapta la vista de Registro de Usuario de Stitch a Angular con estas reglas de negocio:
Regla de acceso:

Clientes pueden registrarse solos desde el login (link "¿No tienes cuenta? Regístrate"). La ruta /registro es pública (sin auth guard). Al registrarse, se crea el usuario con rol cliente automáticamente, sin que el usuario pueda elegir su rol.
Admin, recepcionista y terapeuta NO se pueden registrar solos. Solo el admin puede crearlos desde /terapeutas (nuevo terapeuta) o desde un panel de gestión de usuarios. La vista de Stitch de "Registro" que tienes es en realidad el formulario que usa el admin para crear staff interno, accesible desde dentro del sistema con el guard roles: ['admin'].

Componente público /registro (para clientes):

Archivo: src/app/modules/auth/registro/registro.component.ts|html|scss
Campos: nombre, apellido, email, teléfono, contraseña, confirmar contraseña
NO muestra campo de rol (siempre será cliente)
Al submit llama a POST /api/v1/clientes/ con los datos
Si el backend responde 200, navega a /login con mensaje de éxito
Si el email ya existe, muestra error "Este correo ya está registrado"
Diseño: misma estética del login (card centrada, paleta zen, tipografía Playfair)
Agrega en login.component.html el link: "¿Eres cliente nuevo? Regístrate aquí" → routerLink="/registro"

Componente interno /usuarios/nuevo (para admin, crea staff):

Archivo: src/app/modules/usuarios/nuevo-usuario/nuevo-usuario.component.ts|html|scss
Adapta la vista de Stitch de Registro aquí
Campos: nombre, apellido, email, teléfono, rol (select: admin/recepcionista/terapeuta — NO incluye cliente), contraseña, confirmar contraseña, toggle activo
Validación de contraseña: mínimo 8 caracteres, una mayúscula, un número, un carácter especial (valídalo en el frontend con Validators.pattern)
Al submit llama a POST /api/v1/usuarios/
Ruta protegida con canActivate: [roleGuard], data: { roles: ['admin'] }
Panel informativo lateral (como en el diseño Stitch) explicando cada rol

Genera los tres archivos para cada componente.

PROMPT 7G — Módulos restantes (Clientes, Terapeutas, Servicios, Inventario, Reportes)

[MÓDULOS CRUD VISUALES — CLIENTES, TERAPEUTAS, SERVICIOS, INVENTARIO, REPORTES]
Crea los cinco módulos restantes. Para cada uno sigue el mismo patrón estructural que ves en las capturas de referencia: tabla principal a la izquierda + panel de detalle deslizable a la derecha que aparece al hacer click en un registro.
Patrón base de cada módulo:
scss.module-layout {
display: grid;
grid-template-columns: 1fr 380px; /_ se convierte en 1fr cuando panel cerrado _/
gap: 24px;
transition: grid-template-columns 0.3s ease;
}
.module-layout.sin-panel {
grid-template-columns: 1fr;
}
El panel lateral tiene position: sticky; top: 96px; max-height: calc(100vh - 120px); overflow-y: auto.

CLIENTES (/clientes, ClientesComponent):

Carga: GET /api/v1/clientes/
Header: título, búsqueda, filtros, botón "+ Nuevo cliente"
4 KPI cards: Total clientes, Clientes nuevos (mes), Clientes frecuentes, Valor promedio por cliente (calcula del total de citas)
Tabla: avatar con iniciales, nombre+email, teléfono, fecha nacimiento, última visita (última cita), gasto total (suma de citas completadas), estado activo/inactivo, acciones (ver, editar, eliminar)
Panel detalle al click: tabs Información / Historial / Preferencias. historial_salud solo visible si authService.hasRole('admin', 'terapeuta'). Botones "Editar cliente" y "Eliminar cliente" (soft delete).
Modal de creación/edición con los campos del schema ClienteCreate. Llama a POST o PUT /api/v1/clientes/{id}.

TERAPEUTAS (/terapeutas, TerapeutasComponent):

Carga: GET /api/v1/terapeutas/
4 KPI cards: Total terapeutas, Activos, Citas asignadas hoy (filtra citas de hoy), Calificación promedio (hardcodeado 4.8 por ahora)
Tabla: avatar, nombre+email, especialidad, estado activo, acciones
Panel detalle: tabs Información / Disponibilidad / Rendimiento. Disponibilidad muestra citas de esta semana. Rendimiento muestra ingresos generados del endpoint GET /api/v1/citas/reportes/ingresos-terapeutas.
Modal crear terapeuta: crea usuario+perfil via POST /api/v1/terapeutas/.
Botón desactivar: DELETE /api/v1/terapeutas/{id} (soft delete).

SERVICIOS (/servicios, ServiciosComponent):

Carga: GET /api/v1/servicios/
4 KPI cards: Total servicios, Duración promedio, Precio promedio, Servicio más reservado (del endpoint de reportes)
Tabla: thumbnail placeholder (gradiente por tipo de terapia), nombre+descripción, tipo, duración, precio COP, estado, acciones editar/eliminar
Panel detalle al click: imagen grande, nombre, tipo·duración·precio, tabs Información / Disponibilidad. Información muestra descripción, beneficios (campo hardcodeado por ahora), cabinas compatibles (GET /api/v1/cabinas/{id}/servicios). Botones "Editar servicio" / "Eliminar servicio".
Modal crear/editar: campos del schema ServicioCreate.

INVENTARIO (/inventario, InventarioComponent):

Carga: GET /api/v1/productos/
4 KPI cards: Total productos, Stock total (suma), Bajo stock (cuenta los de stock <= stock_minimo), Valor del inventario (suma de stock \* costo_unitario)
Filtros: búsqueda, dropdown estado (Todos / OK / Bajo stock / Crítico)
Tabla: nombre+descripción, stock con color (verde si ok, naranja si bajo, rojo si 0), stock mínimo, costo unitario COP, valor total (stock×costo), estado badge, acciones
Panel detalle al click: nombre, descripción (hardcodeado por ahora), stock actual, stock mínimo, costo unitario, valor total. Botones "Editar producto" / "Ajustar stock" (abre modal con campo cantidad y radio Entrada / Salida)
Modal ajuste de stock: llama a PUT /api/v1/productos/{id} con el nuevo stock calculado. Valida que el stock no quede negativo en el frontend antes de enviar.

REPORTES (/reportes, ReportesComponent):

Carga: GET /api/v1/citas/reportes/servicios-populares y GET /api/v1/citas/reportes/ingresos-terapeutas
Sección 1 — Servicios más reservados: tabla con ranking, nombre, total reservas, ingresos generados COP. Sin gráfica por ahora (o barras CSS simples con width: X%)
Sección 2 — Ingresos por terapeuta: tabla con nombre, apellido, total citas completadas, ingresos generados COP
Sección 3 — Citas por estado: llama a GET /api/v1/citas/ y agrupa por estado en el frontend. Muestra conteos con los colores de estado de styles.scss.
Filtros de fecha: dos <input type="date"> para fecha_inicio y fecha_fin que pasan como query params a los endpoints

Reglas para TODOS los módulos:

Tres archivos separados por componente: .ts / .html / .scss. Nunca mezcles template en el .ts.
Signals para estado: datos = signal<Tipo[]>([]), seleccionado = signal<Tipo | null>(null), panelAbierto = signal(false), cargando = signal(true)
Precios siempre formateados con Intl.NumberFormat('es-CO', { style: 'currency', currency: 'COP', minimumFractionDigits: 0 })
Fechas formateadas con Intl.DateTimeFormat('es-CO', { day: '2-digit', month: 'short', year: 'numeric' })
Confirmación antes de eliminar: if (confirm('¿Estás seguro de que deseas eliminar este registro?'))
Sin datos hardcodeados, todo viene de la API
Registra todos los componentes nuevos en app.routes.ts

PROMPT 7F — Agenda

[VISTA AGENDA — 3 COLUMNAS CON DATOS REALES]
Adapta la vista de Agenda de Stitch a Angular. Es el módulo más poderoso del sistema.
Archivo: src/app/modules/agenda/agenda.component.ts|html|scss
Layout de 3 columnas fijas:
scss.agenda-grid {
display: grid;
grid-template-columns: 300px 1fr 320px;
gap: 24px;
align-items: start;
height: calc(100vh - 72px - 64px);
}
Columna 1 — Selección de cliente:

Buscador con (input) que filtra llamando a GET /api/v1/clientes/?search={término} con debounce de 300ms (usa Subject + debounceTime)
Lista de clientes: avatar con iniciales, nombre, email, teléfono. Click selecciona el cliente y lo muestra en .info-cliente
Link "+ Nuevo cliente" → navega a /clientes/nuevo
Sección .info-cliente debajo de la lista: muestra foto/avatar, nombre, badge activo, fecha nacimiento, historial de salud (si el rol lo permite), preferencias y link "Ver historial completo"

Columna 2 — Configurar cita:

Campo fecha: <input type="date"> con valor inicial = hoy
Selector terapeuta: cards horizontales con avatar, nombre y especialidad. Llama a GET /api/v1/terapeutas/?activo=true. Click selecciona, card resaltada con border: 2px solid var(--color-primary)
Lista de servicios: llama a GET /api/v1/servicios/. Cada servicio tiene nombre, duración, precio formateado en COP ($ 85,000) y botón + para agregar. Los servicios seleccionados aparecen en un panel .servicios-seleccionados con botón × para quitar
Selector cabina: llama a GET /api/v1/cabinas/?estado=disponible. Cards con imagen placeholder (gradiente), nombre, tipo y badge de estado
Textarea de observaciones con contador de caracteres 0/250

Columna 3 — Resumen de la cita:

Card con position: sticky; top: 96px
Muestra en tiempo real (reactivo con signals): cliente seleccionado, terapeuta, fecha/hora calculada, cabina, lista de servicios con precio individual, subtotal, IVA (19%), total
Botón "Confirmar cita" llama a POST /api/v1/citas/ con el payload correcto. Deshabilitado si falta cliente, terapeuta, cabina o al menos un servicio
Al confirmar con éxito muestra mensaje de confirmación y resetea el formulario
Maneja errores del backend: si responde 409 (terapeuta/cabina ocupada) muestra el mensaje de error en rojo debajo del botón

Botón "Ver calendario" en el header de la vista: por ahora navega a la misma ruta (placeholder para fase futura)
Reglas críticas:

Separa en tres archivos .ts / .html / .scss
Usa signal<Cliente | null>(null) para cliente seleccionado, signal<Terapeuta | null>(null) para terapeuta, signal<Servicio[]>([]) para servicios seleccionados
El total se calcula con computed(() => this.serviciosSeleccionados().reduce(...))
Formatea precios con Intl.NumberFormat('es-CO', { style: 'currency', currency: 'COP', minimumFractionDigits: 0 })
